This rule raises an issue when multiple startswith() or endswith() calls on the same string are chained with the or operator to check for different prefixes or suffixes.

Why is this an issue?

When you need to check if a string starts or ends with one of several possible values, chaining multiple startswith() or endswith() calls with the or operator creates unnecessary verbosity. Both methods accept a tuple of strings as their first argument, so a single call with a tuple is both more concise and more readable.

How to fix it

Replace multiple startswith() or endswith() calls connected by or operators with a single call that uses a tuple containing all the prefixes or suffixes.

Code examples

Noncompliant code example

if s.startswith("http://") or s.startswith("https://"):  # Noncompliant
    process_url(s)

Compliant solution

if s.startswith(("http://", "https://")):
    process_url(s)

Noncompliant code example

if filename.endswith(".jpg") or filename.endswith(".png") or filename.endswith(".gif"):  # Noncompliant
    display_image(filename)

Compliant solution

if filename.endswith((".jpg", ".png", ".gif")):
    display_image(filename)
