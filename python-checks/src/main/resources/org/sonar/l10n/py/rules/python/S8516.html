This rule raises an issue when a group iterator returned by itertools.groupby() is stored and reused after the groupby object has advanced to the next group.

Why is this an issue?

The itertools.groupby() function groups consecutive items from an iterable based on a key function. It returns pairs of (key, group_iterator) where each group_iterator is a sub-iterator that yields items for that specific group.

A critical characteristic of these group iterators is that they are consumed when the main groupby iterator advances to the next group. This means if you store a reference to a group iterator and try to use it later, it will be empty and yield no results.

This behavior occurs because groupby uses lazy evaluation for efficiency. The group iterators share the same underlying data source, and advancing the main iterator invalidates previous group iterators. The problem typically manifests when storing group iterators in a collection for later use or when trying to iterate over the same group iterator more than once.

What is the potential impact?

When group iterators are reused, they yield no items. Your code will process empty sequences instead of the actual grouped data, leading to incorrect results, silent failures, and data loss. These issues can be difficult to debug because the code does not raise exceptions but simply produces wrong output.

How to fix it

Convert the group iterator to a list immediately when processing each group. This materializes the data and stores it in memory, allowing you to reuse it later.

Code examples

Noncompliant code example

from itertools import groupby

data = [1, 1, 2, 2, 3]
groups = {}

for key, group in groupby(data):
    groups[key] = group  # Noncompliant

for key, group in groups.items():
    print(f"{key}: {list(group)}")  # Empty results

Compliant solution

from itertools import groupby

data = [1, 1, 2, 2, 3]
groups = {}

for key, group in groupby(data):
    groups[key] = list(group)  # Convert to list immediately

for key, group in groups.items():
    print(f"{key}: {group}")  # Correct results
